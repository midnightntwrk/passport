What ships

What runs on the screen in October

Seven walkthroughs covering the full account lifecycle, plus a managed-recovery sad-path. The demo runs single-device and uses managed signing through a partner-operated MPC committee — these are the architectural picks the demo accepts; the standards path retires them.

  • Passkey loginFirst visit registers a passkey (Face ID, Touch ID, Windows Hello). Subsequent visits authenticate biometrically. No seed phrase shown at any point.
  • Name registrationUser registers alice.midnight; on-chain confirmation visible. Name uniqueness enforced at the registry.
  • Wallet viewThree role addresses (Shielded, Night, Dust) with balances synced via view key and a hosted indexer.
  • Signing primitiveUser signs arbitrary data and a Night transfer. Signatures verify cross-implementation (TypeScript signer ↔ Rust verifier).
  • Function-call key grantsUser issues, rotates, and revokes scoped function-call keys for dApp authorisation. Multi-device peer behaviour is deferred.
  • Sign-in with PassportA pseudo third-party app requests scoped access; user approves with biometric; transaction signed; function-call key issued.
  • Recovery walkthroughUser "loses" their passkey. Recovery via the auth provider's flow — alt-passkey on a fallback device or backup credential. On-chain account state unchanged; passkey-to-account binding re-established.
