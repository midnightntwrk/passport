Notes for reviewers

  • This is a context + container view. It shows where components live and who they talk to, not the data flow for any specific operation. The runtime view below names the actual modules and walks one end-to-end flow.
  • Workstreams are explicitly unresolved. C3 (DID surface), C4 (asset custody), C22 (intent surface), C24 (fee model), and C25 (cross-chain interface) are dashed because the chosen alternative is not yet decided. Their placement may shift once a workstream resolves.
  • C3 sits inside the on-chain zone as the primary architectural element, but the workstream's resolution runs through engagement with the external midnight-did repository — drawn alongside in the external identity ecosystem.
  • C8 (domain separation) is cross-cutting and shown as an annotation rather than placed in any single zone — every persistentHash use site needs a registered prefix.
  • C6 / C7 / C20 are placed in the proof server sub-zone for clarity; in practice their orchestration code lives in the wallet runtime. The trust property is what matters: witnesses must not leave the user device.
  • C26 (AI agent skills) is omitted — it's a meta-deliverable about documentation and tooling, not a runtime element.
  • The MVP vs. v1.0 split is not represented here; this view holds for both readings. Standards and Demo document where they diverge.

Identity, signing & custody conceptual · open relationships marked TBD
